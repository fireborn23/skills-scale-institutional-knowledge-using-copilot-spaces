# Process Doc Update — Review & Acceptance Checklist

Purpose: Ensure proposed changes to program/process documentation are reviewed, aligned with stakeholders, and tracked to completion.

How to use: Add this checklist to PRs that change docs/* process files or use it as a template in issue comments.

- Title: Short summary of the proposed doc change
- Affected document(s): e.g., docs/octoacme-roles-and-personas.md
- Summary of change: One-paragraph description and rationale
- Acceptance criteria (required):
  - [ ] Content aligns with existing process docs
  - [ ] Update improves clarity or closes a documented gap
  - [ ] Responsibilities, handoffs, and owners are clearly defined
  - [ ] Communication & escalation paths are documented (if applicable)
- Stakeholder review:
  - [ ] Product Manager (or owner) reviewed
  - [ ] Project Manager (or owner) reviewed
  - [ ] (Optional) Security / Compliance reviewed (if relevant)
  - [ ] (Optional) Support / Ops reviewed (if relevant)
- Implementation notes:
  - [ ] Add references to related docs (links)
  - [ ] Update project README or .copilot/ context if needed
- Follow-up:
  - [ ] Assign follow-up action items (owner + due date)
  - [ ] Add to weekly PM sync agenda (if decision required)
- PR metadata:
  - Linked issue: #4
  - Suggested reviewer(s): fireborn23
  - Branch: add/personas-process-improvements-4

Example usage:
- Add this checklist to the PR description or as an attached file.
- Tick the stakeholder review boxes as reviews are completed.
